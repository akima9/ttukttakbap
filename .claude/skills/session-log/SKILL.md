# 세션로그 작성

뚝딱밥 프로젝트의 작업 단위가 끝날 때마다 세션로그를 남긴다.

## 언제 실행하나
- 하나의 **작업/기능 단위**(기능 구현, 버그 수정, 리팩터링, 설계 결정 등)가 완료되어 사용자에게 "완료" 보고를 하는 시점.
- 사용자가 `/session-log` 또는 "세션로그 남겨줘"라고 요청할 때.

여러 작업을 한 번에 끝냈다면 작업별로 블록을 나눠 기록한다.

## 어디에 쓰나
디렉토리(절대경로, 레포 밖 Obsidian 볼트):
```
/Users/kimgy/Documents/ObsidianVault/프로젝트/뚝딱밥/세션로그/
```
파일명은 **그 날짜 하루 1개**: `YYYY-MM-DD.md` (예: `2026-06-28.md`).

- 오늘 날짜 파일이 **없으면** 새로 만들고, 맨 위에 `## YYYY-MM-DD 세션` 헤더를 둔다.
- 이미 **있으면** 파일 맨 아래에 이번 작업 블록만 **이어붙인다**(기존 내용 보존, 덮어쓰기 금지). 먼저 Read로 기존 내용을 읽고 Edit로 끝에 추가한다.

## 작업 블록 형식
하나의 작업 = 하나의 `### ` 블록. 빈 항목은 생략한다.

```markdown
### [작업명] (HH:MM)
- **완료한 것**: 무엇을 했는지 한 줄씩
- **주요 결정**: 왜 그렇게 했는지(트레이드오프·근거). 없으면 생략
- **변경/커밋**: 핵심 파일과 커밋 해시(있으면)
- **발생한 문제**: 문제 / 원인 / 해결 / 레벨(1~3). 없으면 생략
- **다음에 할 것**: 이어서 할 작업
```

## 규칙
- 사실만 적는다. 테스트가 실패했거나 건너뛴 단계가 있으면 그대로 적는다.
- 짧고 검색 가능하게. 의미 있는 요약(무엇을·왜)을 남기고 코드 전체를 붙여넣지 않는다.
- 작성 후 사용자에게 "세션로그 남김: `파일명` → `작업명`" 한 줄로 알린다.
