# 장보기 체크리스트 재료 행(Row) — UI 스펙

> 작성일: 2026-07-17 · 상태: 초안 · 슬러그: 02-ingredient-row · 다음 담당: frontend-dev

재료 체크리스트 행 1개(`IngredientsChecklist.tsx`의 `ordered.map` 내부 `<li>`)의 레이아웃·쿠팡 보조 버튼·상태별 비주얼 스펙.
기능·정책·우선순위는 [`../planning/02-ingredient-row.md`](../planning/02-ingredient-row.md) 참조(중복 서술 안 함). **문서 스펙이며 코드 구현은 frontend-dev 몫.**

## 0. 전제 · 토큰 현황 (근거)

- `globals.css`에 커스텀 디자인 토큰 없음(`--background`/`--foreground`뿐) → 모든 값은 **Tailwind 기본 유틸리티**로 지정. 신규 `@theme` 토큰 추가 없음.
- 기존 팔레트 유지: `rose-50 #fff1f2` / `rose-200 #fecdd3` / `rose-500 #f43f5e` / `rose-600 #e11d48`, `gray-50 #f9fafb` / `gray-100 #f3f4f6` / `gray-200 #e5e7eb` / `gray-300 #d1d5db` / `gray-400 #9ca3af` / `gray-500 #6b7280` / `gray-600 #4b5563` / `gray-800 #1f2937`.
- 데이터는 `name` + `requiredAmount`+`unit` + `coupangUrl`(null 가능) + 로컬 체크상태가 전부(planning §2 확정). 추가 없음.
- 라벨 "쿠팡" 사용 가능 — 직전까지 "쿠팡에서 보기" 텍스트 링크를 운영한 실적이 있음(planning §7-1 미확인 항목은 이로써 해소).

## 1. 권고안 결정

### 채택: **중립 톤 아웃라인 칩(아이콘 + "쿠팡" 라벨)**

쿠팡 링크를 **라벨 있고 · 경계 있고 · 색 무게는 없는** 보조 칩으로 바꾼다. 진단 3건을 한 번에 푼다.

| 진단 | 해소 방식 |
|------|-----------|
| #1 의미 공백 | 가시 라벨 **"쿠팡"** + **외부 이동 아이콘**(장바구니 아이콘 폐기, §3-1) |
| #2 경계 부재 · 무게 역전 | 상시 `border-gray-200 bg-gray-50` 경계 부여 + 시각 높이 44 → **36px로 축소**, 탭 타깃은 별도 확보(§3-2) |
| #3 두 빨강 충돌 | 버튼에서 **빨강 제거**(중립 회색). 필요량만 빨강을 독점(§4) |

### 기각: 미니멀 대안(라벨 없이 경계 + 크기 축소만) — planning §7-3

기각 근거 3가지.
1. **문제의 1순위를 안 푼다.** 기획서가 #1(의미 공백)을 심각도 '높음'으로 짚었고, 미니멀 대안은 스스로 "#1은 남는다"고 인정한다. 장바구니 아이콘의 관습("내 장바구니")과 실제 동작(외부 이탈)의 충돌은 경계를 줘도 그대로 남는다 — 오히려 **버튼처럼 보이게 만들면 "담기 버튼"으로 오독될 위험이 커진다.**
2. **수용 기준 1을 통과 못 한다.** "라벨만으로 쿠팡 이동임을 알 수 있다"(planning §8-1)는 라벨 없이 충족 불가.
3. **수익화에 불리하다.** 라벨 없는 아이콘은 클릭 전 목적지를 모르게 해 클릭을 억제한다. 라벨은 노출을 줄이지 않으면서 명료도만 올린다(planning §3-Q3 부수효과).

라벨의 비용(폭 +~40px)은 우측 열에서만 발생하고, 360px에서도 여유가 있다(§6 계산).

## 2. 행 레이아웃

```
<li>  px-4 py-3 · flex items-center justify-between · gap-3
├─ 좌 그룹  flex items-center gap-3 · min-w-0        ← 남는 폭 전부, 줄바꿈 허용
│   ├─ 체크 원   w-5 h-5 shrink-0
│   └─ 재료명    (min-w-0, break-keep)
└─ 우 그룹  flex items-center gap-3 · shrink-0       ← 폭 고정, 절대 안 줄어듦
    ├─ 필요량   min-w-[3.5rem] text-right
    └─ 쿠팡 칩  w-16 h-9   (또는 동일 폭 빈 자리)
```

- 좌 그룹에 `min-w-0`을 반드시 준다 — 없으면 flex 자식이 축소되지 않아 긴 재료명이 우측 그룹을 밀어낸다.
- 우 그룹은 `shrink-0` 유지(planning §6: 필요량·버튼은 절대 잘리지 않는다).
- 그룹 간 간격: `gap-3`(12px). 필요량↔칩 간격도 `gap-3`으로 현행 `gap-2`(8px)에서 넓힌다 — 기획 진단 #3의 "8px로 붙어 뭉친다"를 색이 아닌 **간격 축**에서도 함께 완화.

### 행 높이 (변하지 않아야 함 — planning §8-5)

| | 현재 | 신규 |
|---|---|---|
| 우측 액션 기여 높이 | 44 − 16(`-my-2`) = **28px** | 36 − 8(`-my-1`) = **28px** |
| 콘텐츠 높이 | 28px | 28px |
| 행 총 높이 | 28 + 24(`py-3`) = **52px** | **52px (동일)** |

칩의 시각 높이는 36px지만 `-my-1`로 상하 4px씩 `py-3`(12px) 패딩 안으로 흡수 → **행 높이 회귀 없음.**

## 3. 쿠팡 보조 칩 스펙

### 3-1. 형태 · 아이콘 · 라벨

| 항목 | 값 | 근거 |
|------|-----|------|
| 형태 | 아웃라인 칩(라운드 사각) `rounded-lg` | 행 카드 `rounded-xl`보다 한 단계 작게 → 부속 요소임을 형태로 표현. pill(`rounded-full`)은 태그/필터로 읽혀 액션성이 약해짐 → 미채택 |
| 라벨 | **`쿠팡`** | 목적지를 정확히 말함. `주문`은 앱 내 주문 오해, `쿠팡에서 보기`는 360px에서 폭 과다 |
| 아이콘 | **외부 이동(external-link)** `w-3.5 h-3.5`, `strokeWidth={2}`, `aria-hidden="true"` | **장바구니 아이콘 폐기.** 관습("내 장바구니")과 실제 동작(외부 이탈)이 충돌한다는 진단 #1의 직접 해법. 외부 이동 아이콘은 "이 링크는 밖으로 나간다"는 실제 의미와 일치하고 `target="_blank"`의 시각적 예고가 됨 |
| 아이콘 위치 | 라벨 **뒤**(오른쪽) | 읽기 순서 "쿠팡 → (밖으로)". 외부 링크 표기 관습 |
| 크기 | 시각 `w-16 h-9` (64×36px) 고정 | 폭 고정이 planning §8-6(링크 없는 재료 섞여도 열 안 어긋남)의 전제. 내부 계산: `px-2.5`(20) + "쿠팡" 12px 2자(24) + `gap-1`(4) + 아이콘(14) = 62 ≤ 64 |
| 타이포 | `text-xs font-medium` (12px/500) | 재료명 16px/500, 필요량 16px/600보다 확실히 아래 → 위계 역전 방지(planning §8-2) |

아이콘 path(heroicons outline, external-link):
`M10 6H6a2 2 0 00-2 2v10a2 2 0 002 2h10a2 2 0 002-2v-4M14 4h6m0 0v6m0-6L10 14`

### 3-2. 44×44 탭 타깃 확보 방식

시각 높이는 36px지만 탭 타깃은 44px를 지켜야 한다(planning §4-3). **현행 음수 마진(`-my-2`)으로 44px 요소를 우겨넣는 방식은 폐기**한다 — 그게 바로 "면적만 최대"인 진단 #2의 원인이기 때문이다. 대신 **시각 크기와 히트 영역을 분리**한다.

```
relative -my-1
before:absolute before:content-[''] before:inset-x-0 before:-inset-y-1
```

- `before:-inset-y-1`: 위아래 4px씩 확장 → 히트 영역 **64 × 44px** (44×44 초과 충족).
- 확장된 4px는 `li`의 `py-3`(12px) 패딩 안이라 인접 행을 침범하지 않는다.
- 의사요소는 `<a>`의 히트 영역이므로 `stopPropagation`이 그대로 걸린다 → 확장 영역 탭도 체크 토글 안 됨(planning §8-4).
- `-my-1`은 §2의 행 높이 유지용(28px 기여).

### 3-3. 링크 유지 속성 (현행 유지)

`target="_blank"` · `rel="noopener noreferrer"` · `onClick={e => e.stopPropagation()}` · `aria-label={`${name} 쿠팡에서 보기`}`.
가시 라벨("쿠팡")과 aria-label이 다르지만 **가시 라벨이 aria-label에 부분 문자열로 포함**되므로 WCAG 2.5.3(Label in Name) 충족.

## 4. 두 빨강 충돌 해소 — 빨강은 필요량이 독점한다

### 결정: **버튼에서 빨강을 뺀다(→ 중립 회색). 필요량은 빨강을 유지하되 `rose-600`으로 승격.**

| 안 | 내용 | 판정 |
|----|------|------|
| A | 필요량을 `gray-800`로 낮추고 버튼에 쿠팡 레드 유지 | **기각.** 필요량은 인원 스테퍼의 결과가 드러나는 유일한 지점(planning §3-Q1) = 이 화면의 주 정보. 주 정보를 죽이고 부 액션에 브랜드 색을 넘겨주면 위계가 뒤집힌다(planning §8-2 위반) |
| B | **필요량 빨강 유지(`rose-600`) + 버튼 중립 회색** | **채택** |
| C | 둘 다 유지하고 간격/구분선으로만 분리 | **기각.** 8px→12px로 벌려도 같은 계열 두 빨강이 한 시선 안에 남아 "미세하게 어긋남"이 해소되지 않음. 단, 간격 확대는 B와 함께 보조 수단으로 병행(§2) |

**근거**
1. **빨강은 이 앱에서 "강조" 신호다**(체크 원 채움, 진행바, CTA, 완료 배너). 부 액션이 같은 신호를 쓰면 신호 자체가 인플레된다. 버튼을 중립화하면 색 충돌(#3)과 무게 역전(#2)이 **한 번에** 풀린다.
2. **브랜드 색 없이도 브랜드는 전달된다.** 목적지 식별은 이제 `#C0392B`가 아니라 **"쿠팡" 라벨**이 담당한다 — 색보다 정확하고 색맹 사용자에게도 동작한다. 브랜드 레드는 라벨이 없던 시절의 유일한 단서였기에 필요했던 것이고, 라벨을 얻은 지금은 역할이 끝났다.
3. **`#C0392B`는 어차피 우리 팔레트에 없는 외래값**이다(Tailwind 임의값 `text-[#C0392B]`). 이 값을 지우면 컴포넌트의 색은 rose/gray 두 축으로 정리된다 — 토큰 최소화 원칙과 일치.

### 필요량 `rose-500` → `rose-600` 승격 (접근성 수정 동반)

- `rose-500 #f43f5e` on 흰 배경 = **3.55:1** → 16px/600은 WCAG 대형 텍스트(18.66px bold 이상)가 아니므로 **AA 4.5:1 미달**(현행 결함).
- `rose-600 #e11d48` on 흰 배경 = **4.69:1** → **AA 통과**.
- 부수 효과: 웜톤 쪽으로 살짝 이동해 원래 `#C0392B`와 부딪히던 쿨한 핑크레드 성격도 완화된다(다만 버튼이 중립화되므로 이 효과는 보너스).
- **적용 범위는 필요량 텍스트만.** 진행바 채움·체크 원·CTA의 `rose-500`은 그래픽/대형 요소라 3:1 기준을 이미 만족하므로 **건드리지 않는다**(01-shopping-progress와의 일관성 유지, 변경 최소화).

## 5. 상태별 스펙

### 5-1. 쿠팡 칩

공통 베이스:
`relative -my-1 w-16 h-9 shrink-0 inline-flex items-center justify-center gap-1 rounded-lg border text-xs font-medium transition-colors before:absolute before:content-[''] before:inset-x-0 before:-inset-y-1`

| 상태 | 조건 | 추가 클래스 | 값 |
|------|------|-------------|-----|
| **기본(미체크 행)** | `coupangUrl && !isChecked` | `border-gray-200 bg-gray-50 text-gray-600` | 테두리 `#e5e7eb` / 배경 `#f9fafb` / 텍스트 `#4b5563` — hover 없이도 경계가 보임(planning §8-3, 터치 기기 대응) |
| **hover** | 포인터 기기 | `hover:bg-gray-100 hover:border-gray-300 hover:text-gray-800` | 배경 `#f3f4f6` / 테두리 `#d1d5db` / 텍스트 `#1f2937` |
| **active** | 누르는 중 | `active:bg-gray-200` | `#e5e7eb` (터치 피드백) |
| **focus-visible** | 키보드 포커스 | `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-rose-500 focus-visible:ring-offset-2` | 링 `#f43f5e`, 오프셋 2px. 앱의 CTA 포커스 관습(`ring-rose-300 ring-offset-2`)과 같은 계열 — **색이 아니라 링 자체**가 신호이므로 중립 칩 위에 브랜드 링을 써도 §4 결정과 충돌 없음 |
| **체크됨(dimmed)** | `coupangUrl && isChecked` | `border-gray-200 bg-transparent text-gray-600` | **배경 채움만 제거**해 rose-50 행에 녹아들며 물러난다. 클릭 가능·DOM 유지·폭 고정(planning §4-5). hover/focus는 기본과 동일하게 살아 있음 |
| **링크 없음** | `coupangUrl === null` | `<span className="w-16 shrink-0" aria-hidden="true" />` | 칩을 렌더하지 않고 **동일 폭 64px만 예약**. 필요량 우측 정렬 열 유지(planning §8-6). 높이 미점유(`-my-1` 불필요) |

**체크됨 상태에서 텍스트를 `gray-400`로 낮추지 않은 이유**: `gray-400 #9ca3af` on `rose-50` = **2.31:1**로 AA(4.5:1) 크게 미달이고, `gray-500 #6b7280` on `rose-50`도 **4.40:1**로 여전히 미달이다. 이 칩은 **비활성이 아니라 계속 클릭 가능한 링크**이므로 WCAG의 disabled 예외가 적용되지 않는다. 따라서 **dim은 텍스트 색이 아니라 배경 채움(fill)에서 뺀다** — 시각 무게는 충분히 내려가면서(`filled chip` → `ghost chip`) 대비는 `gray-600` on `rose-50` = **6.80:1**로 안전하다.

### 5-2. 행 · 좌측 · 필요량

| 요소 | 미체크 | 체크됨 |
|------|--------|--------|
| `li` | `bg-white border-gray-100 shadow-sm rounded-xl` | `bg-rose-50 border-rose-200` (현행 유지) |
| 체크 원 | `w-5 h-5 rounded-full border-2 border-gray-300` | `border-rose-500 bg-rose-500` + 흰 체크 svg (현행 유지) |
| 재료명 | `font-medium text-gray-800` | `line-through text-gray-600` ← **`gray-400`에서 변경 권고, §7-1** |
| 필요량 | `font-semibold min-w-[3.5rem] text-right` **`text-rose-600`** ← `rose-500`에서 변경(§4) | `font-semibold text-gray-600` ← **`gray-400`에서 변경 권고, §7-1** |
| 쿠팡 칩 | filled | ghost (5-1) |

## 6. 반응형 (모바일 우선)

360px 폭 기준 실측 계산:

```
화면 360 − 페이지 좌우 여백(px-4 ×2 = 32)            = 328   ← li 바깥 폭
li 내부 = 328 − px-4 ×2(32)                          = 296
우 그룹 = 필요량 56 + gap-3 12 + 칩 64               = 132   (shrink-0, 고정)
그룹 간 gap-3                                        =  12
좌 그룹 = 296 − 132 − 12                             = 152
재료명 가용 = 152 − 체크원 20 − gap-3 12             = 120px
```

- 재료명 120px ≈ 한글 16px 기준 **7~8자**. 그 이상은 **줄바꿈**한다.
- 줄바꿈 규칙: `break-keep`(한글 어절 단위로 끊어 낱글자 분리 방지). `truncate` 미채택 — 재료명은 마트에서 찾는 검색 키(planning §3-Q1 Must)라 **가리면 안 된다**. 줄바꿈 시 해당 행만 높이가 늘고, 우 그룹은 `items-center`로 세로 중앙 유지.
- 필요량 `min-w-[3.5rem]`(56px)은 최장값 `"1000ml"`(16px/600 ≈ 52px)를 수용 → 잘림 없음(planning §8-7).
- 브레이크포인트 분기 없음. 360px에서 성립하면 그 이상은 좌 그룹만 넓어진다.

## 7. 접근성

**⚠ 현행 결함 2건 수정 권고** (둘 다 이 행 범위 안이지만 planning에 명시되지 않은 항목 → §8 열린 결정):
1. 필요량 `rose-500` → `rose-600` (3.55 → 4.69). §4에서 이미 채택.
2. 체크된 재료명·필요량 `gray-400` → `gray-600` (2.31 → 6.80). 체크 상태는 이미 **체크 원 채움 + 취소선 + 행 배경 rose-50** 3중 단서로 전달되므로(planning §9 NN/g 근거) **색 dim은 부하가 없는 장식**이다. 색을 한 단계만 낮춰도 상태 전달에 손실이 없고 대비만 회복된다.

### 7-2. 색에만 의존하지 않기 (1.4.1)

- 체크 상태: 원 채움(형태) + 취소선(형태) + 배경(색) + `aria-checked` → **색 제거해도 판별 가능**.
- 쿠팡 액션: 라벨 텍스트 "쿠팡"(문자) + 외부 아이콘(형태) + 테두리(형태) → **색 제거해도 판별 가능**. 브랜드 레드 폐기가 접근성에 손해를 주지 않는 이유가 이것.

### 7-3. 탭 타깃 · 포커스

- 쿠팡 칩: 히트 64×44 (§3-2). 행 전체: 328×52 — 둘 다 44px 축 충족.
- 두 인터랙티브 영역이 겹치므로 **칩의 `stopPropagation`은 필수 유지**.

### 7-4. 키보드 접근 (planning §7-2 열린 결정 → **이번 범위에 포함 권고**)

현재 `<li onClick>`은 포커스 불가라 **키보드/스크린리더 사용자는 체크를 아예 못 한다** — 화면의 주 액션이 통째로 막힌 상태다. 쿠팡 칩(`<a>`)만 도달 가능해서, 고치지 않으면 "부 액션만 조작 가능한 행"이 된다. 이번에 함께 처리할 것을 권고한다.

디자이너가 정의하는 부분은 **포커스 링 비주얼**뿐이고, 접근 가능한 구조(role·키 핸들링)는 frontend-dev 판단:
- 행 포커스 링: `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-rose-500 focus-visible:ring-offset-2` (칩과 동일 관습, 행 `rounded-xl` 따라감)
- 포커스 순서: 행 → 칩 (DOM 순서와 일치, `tabIndex` 조작 없이)
- 참고 구조: `role="checkbox"` + `aria-checked={isChecked}` + `tabIndex={0}` + Space/Enter 핸들링

## 8. 열린 결정 (사용자/메인 확인 필요)

1. **§7-1 결함 2건 수정 여부** — 필요량 `rose-500`→`rose-600`(§4 채택안에 포함), 체크된 텍스트 `gray-400`→`gray-600`. 둘 다 planning에 없던 항목이고 "체크 행이 덜 흐려 보인다"는 인상 변화가 있다. AA 준수를 택할지 현행 dim 강도를 택할지 확인 필요. **디자이너 권고: 수정.** 색 dim은 이 행에서 중복 신호라 잃는 것이 없다.
2. **키보드 접근(§7-4)을 이번 슬러그에 포함할지** — 권고: 포함(주 액션이 막혀 있음). 분리한다면 별도 슬러그로 즉시 큐잉.
3. **아이콘 교체(장바구니 → 외부 이동)** — 진단 #1의 핵심 처방이라 채택을 전제로 썼으나, "쿠팡=장바구니"라는 기존 시각 자산을 버리는 변경이므로 명시적 확인이 필요.
