# 장보기 진행률 표시 — 요구사항 정의

> 작성일: 2026-07-17 · 상태: 초안 · 슬러그: 01-shopping-progress · 다음 담당: web-designer

재료 체크리스트 화면에서 사용자가 담은(체크한) 재료 수를 전체 대비로 보여주는 **순수 UI 기능**의 요구사항을 정의한다. 백엔드/DB 변경 없음.

## 1. 배경 / 목표

- **문제**: 재료가 많으면 "지금 어디까지 담았는지"를 사용자가 한눈에 알기 어렵다. 진행 상황 피드백이 없어 장보기 완료 여부를 스스로 세어야 한다.
- **목표**: 체크리스트 상단에 "담은 수 / 전체 수 + 진행바"를 표시해 장보기 진행 상황을 즉시 인지하게 한다.
- **성공 기준**: 재료를 체크/해제할 때마다 진행 텍스트와 진행바가 실시간으로 반영된다.

## 2. 현황 (코드 확인)

- 파일: `frontend/src/app/ingredients/IngredientsChecklist.tsx`
- `checked: Set<number>` 상태로 체크된 재료 ID를 관리하고, `toggle()`로 항목을 켜고 끈다.
- 전체 재료는 `ingredients: Ingredient[]` prop으로 들어온다.
- 따라서 **담은 수 = `checked.size`**, **전체 수 = `ingredients.length`**. 새 상태나 API 없이 파생값만으로 구현 가능하다.

## 3. 기능 요구사항 (MoSCoW)

| 기능 | 설명 | 우선순위 | 비고 |
|------|------|----------|------|
| 진행 텍스트 | "3/6 담음" 형태로 담은 수 / 전체 수 표시 | Must | `checked.size` / `ingredients.length` |
| 진행바 | 담은 비율(0~100%)을 채우는 가로 바 | Must | 체크/해제 시 실시간 갱신 |
| 배치 | 체크리스트(`<ul>`) 바로 위, 목록 복사·공유 버튼 영역과 목록 사이 | Must | 스크롤 없이 보이는 위치 |
| 완료 강조 | 전부 담으면(전체=담은 수) 진행바/문구를 완료 상태로 강조 | Should | 예: "장보기 완료!" |
| 상태 전환 애니메이션 | 진행바 채움에 부드러운 전환 | Could | 과하지 않게 |

## 4. 범위

**In (포함)**
- `IngredientsChecklist.tsx` 내부의 진행률 텍스트 + 진행바 UI 추가.
- 기존 `checked` / `ingredients`에서 파생값 계산.

**Out (제외)**
- 백엔드/DB/API 변경.
- 체크 상태의 서버 저장·영속화(새로고침 시 초기화되는 현행 동작 유지).
- 재료 정렬·필터, "담은 항목 숨기기" 등 신규 인터랙션.
- 다른 화면(메뉴·레시피)으로의 진행률 전파.

## 5. 정책 / 엣지 케이스

- **재료 0개**(`ingredients.length === 0`): 진행률 영역을 표시하지 않는다(0/0·바 미표시). 현재 페이지 특성상 드물지만 방어한다.
- **비율 계산**: `전체가 0`이면 나눗셈을 피한다. 그 외 `담은 수 / 전체 수 × 100`.
- 진행바는 값이 0일 때도 트랙(빈 바)이 보여 위치를 인지시킨다.

## 6. 수용 기준 (Acceptance)

1. 재료 6개 중 3개 체크 시 "3/6 담음"과 50% 채워진 진행바가 보인다.
2. 항목을 추가로 체크/해제하면 텍스트와 바가 즉시 갱신된다.
3. 전체를 체크하면 완료 상태(문구/바 강조)로 바뀐다.
4. 재료가 0개면 진행률 영역이 렌더되지 않는다.
5. 기존 목록 복사·공유·레시피 이동 동작에 영향이 없다.

- 팔레트는 기존 화면과 통일(체크 상태에 `rose-500` 계열 사용 중).
